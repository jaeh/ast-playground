Welcome to the magic docs.

The goal of this document is to give you a rough @magical overview.

Features

client app

magic uses hyperapp to generate a client side webapp.

since hyperapp is awesomely small, the minimal client bundle size hovers around 3-4kb (gzipped), this includes all of @magic and depends on the variety of html elements in use as well as the amount and content length of the pages.

static file hosting:

publishes to github, gitlab and any other git enabled hosting service.

serverless / faas

automagically generates serverless lambdas, derived from the @magic-modules you use in your pages. this makes visitor statistics, user authentication and authorization, chat, and all other server side services possible.
